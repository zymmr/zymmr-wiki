Managing Project Backlog in Zymmr

Effectively managing your project backlog in Zymmr involves understanding how to add, edit, and organize work items, as well as how to move them into sprints for active work. Here's a detailed guide on managing your project backlog:

Key Functionalities​

Backlog Overview​

  • Displayed Fields: Each backlog item is shown with the following fields:
    • Type
    • Title
    • Project
    • Status
    • Assignee
    • Story Point
    • Priority
  • Editable Fields: Type, Title, and Story Point can be added and edited.
  • Read-Only Fields: Project, Status, and Priority are read-only.

Backlog Items Count​

The number of items in the backlog is displayed next to the backlog section.

Sprint Management​

  1. Creating Sprints: Click on the "Create Sprint+" button at the bottom of the backlog to create a new sprint.
  2. Sprint Display: Once created, sprints appear at the top of the backlog. Multiple sprints are displayed at the top for easy access.
  3. Moving Tasks: Tasks can be moved between the backlog and sprints to organize work items according to their priority and readiness.

Filtering Work Items​

You can filter work items by project or multiple projects to display only relevant items in the backlog.

Example Use Case​

Scenario​

You need to organize and manage the backlog for the "Mobile App Development" project, ensuring that tasks are properly prioritized and ready for upcoming sprints.

Steps​

  1. View Backlog: Access the backlog to see all the items listed with their type, title, project, status, assignee, story point, and priority.
  2. Edit Items: Add or edit the type, title, and story point for any backlog items as needed.
  3. Create Sprint: Click the "Create Sprint+" button at the bottom of the backlog to create a new sprint named "Sprint 1."
  4. Move Tasks to Sprint: Drag and drop relevant tasks from the backlog into "Sprint 1" to prepare them for active work.
  5. Filter Work Items: Use the filter option to display only the "Mobile App Development" project work items in the backlog for focused management.
  6. Review and Adjust: Periodically review the backlog and sprints, making adjustments to ensure tasks are prioritized correctly and moved to sprints as they become ready.

By effectively managing your project backlog in Zymmr, you can ensure that tasks are organized, prioritized, and ready for your team to work on, leading to better project planning and execution.
